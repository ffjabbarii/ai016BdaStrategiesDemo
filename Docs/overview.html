Project Overview

Architecture and Implementation Strategy

Business Document Analysis (BDA) Strategies

This project demonstrates three different approaches to processing business documents using AWS services, implemented in both Python and C#.

Architecture Overview

Document Upload → AWS Services → Processing Engine → Structured Data Output Three Processing Patterns: 1. Blueprint API: Template-based extraction 2. Textract Direct: Raw OCR with custom parsing 3. AnalyzeDocument: Enhanced ML-powered analysis

Supported Document Types

  • W-2 Forms: Tax documents with structured fields
  • Bank Statements: Financial documents with transaction tables

Implementation Languages

Python Implementation

  • FastAPI for REST APIs
  • boto3 for AWS integration
  • OpenCV for image processing
  • Lambda deployment

C# Implementation

  • ASP.NET Core for REST APIs
  • AWS SDK for .NET
  • ImageSharp for image processing
  • Lambda and ECS deployment
