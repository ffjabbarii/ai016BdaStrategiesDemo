üîç The Problem We Solved

‚ùå Original Issue

The AWS BDA (Bedrock Data Automation) API was failing with:

Member must satisfy regular expression pattern: 
arn:aws(|-cn|-iso|-iso-[a-z]|-us-gov):bedrock:[a-zA-Z0-9-]*:(aws|[0-9]{12}):data-automation-profile/[a-zA-Z0-9-_.]+

Root Cause: The code was incorrectly using the project ARN as the profile ARN:

# WRONG - This was causing the error

‚úÖ The Fix

Implemented proper profile ARN resolution with multiple AWS patterns:

# CORRECT - Now resolves proper profile ARN profile_arn = await self._get_or_create_data_automation_profile(project_arn) dataAutomationProfileArn=profile_arn

async def _get_or_create_data_automation_profile(self, project_arn: str) -> str:
    """Get or create the correct data automation profile ARN for BDA processing"""
    
    # Extract account ID and region from project ARN
    arn_parts = project_arn.split(':')
    region = arn_parts[3]
    account_id = arn_parts[4]
    
    # Try different profile ARN patterns based on AWS documentation
    profile_candidates = [
        f"arn:aws:bedrock:{region}:{account_id}:data-automation-profile/default",
        f"arn:aws:bedrock:{region}:aws:data-automation-profile/default",
        f"arn:aws:bedrock:{region}:{account_id}:data-automation-profile/standard",
        f"arn:aws:bedrock:{region}:{account_id}:data-automation-profile/{project_arn.split('/')[-1]}"
    ]
    
    # Test each candidate and return the first working one
    for candidate_arn in profile_candidates:
        # Validation logic here...
        return candidate_arn
