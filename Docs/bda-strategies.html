What you want

Drop a PDF (W-2 or bank statement) into a workflow and get structured output back (JSON) plus a portal to browse results.

Do you need to create a blueprint every time?

  • No. Create a blueprint once per document type and reuse it.
  • Create a new blueprint only when layouts differ a lot (e.g., different bank formats).

Five practical flows

Recommended default production flow

S3 upload (raw) → trigger (Lambda/Step Functions) → extraction → normalize → store results → publish to portal

Outputs to standardize

  • Raw extracted JSON
  • Normalized JSON (your schema)
  • Optional HTML summary per doc
