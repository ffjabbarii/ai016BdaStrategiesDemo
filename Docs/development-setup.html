Development Setup

VS Code and Visual Studio Configuration

Development Environment Setup

This guide helps you set up VS Code and Visual Studio for optimal development experience with both Python and C# implementations.

Prerequisites

  • Git: Version control system
  • AWS CLI: AWS command line interface
  • Python 3.8+: For Python implementations
  • .NET 6 SDK: For C# implementations
  • Docker: For containerized deployments

Visual Studio Setup

Required Workloads

  • ASP.NET and web development: For web APIs
  • Azure development: For cloud deployments
  • .NET desktop development: For console applications
  • Python development: For Python projects (Visual Studio 2022)

Solution Structure


# Create solution file for C# projects
dotnet new sln -n BdaStrategies

# Add projects to solution
dotnet sln add csharp/BlueprintAPI/BlueprintAPI.csproj
dotnet sln add csharp/Textract/Textract.csproj  
dotnet sln add csharp/AnalyzeDocument/AnalyzeDocument.csproj

# Add test projects
dotnet sln add tests/BlueprintAPI.Tests/BlueprintAPI.Tests.csproj

Python Environment Setup


# Create virtual environment
cd python
python -m venv venv

# Activate virtual environment
# Windows
venv\Scripts\activate
# macOS/Linux  
source venv/bin/activate

# Install dependencies for each project
cd BlueprintAPI && pip install -r requirements.txt
cd ../Textract && pip install -r requirements.txt
cd ../AnalyzeDocument && pip install -r requirements.txt

AWS Configuration


# Configure AWS CLI
aws configure
# AWS Access Key ID: [Your Access Key]
# AWS Secret Access Key: [Your Secret Key]  
# Default region name: us-east-1
# Default output format: json

# Verify configuration
aws sts get-caller-identity

# Set up AWS profiles (optional)
aws configure --profile dev
aws configure --profile prod

Git Configuration


# Clone repository
git clone https://github.com/ffjabbarii/ai016BdaStrategiesDemo.git
cd ai016BdaStrategiesDemo

# Set up Git hooks (optional)
cp .githooks/pre-commit .git/hooks/
chmod +x .git/hooks/pre-commit

# Configure Git
git config user.name "Your Name"
git config user.email "your.email@example.com"

Build and Test Workflow

Python Projects


# Navigate to Python project
cd python/BlueprintAPI

# Install dependencies
pip install -r requirements.txt

# Run locally
python src/api.py

# Run tests
pytest tests/

# Format code
black src/
pylint src/

C# Projects


# Navigate to C# project
cd csharp/BlueprintAPI

# Restore dependencies
dotnet restore

# Build project
dotnet build

# Run locally
dotnet run

# Run tests
dotnet test

# Format code
dotnet format

Debugging Configuration

VS Code Debugging

  • Set breakpoints in Python/C# code
  • Use F5 to start debugging
  • Inspect variables in Debug Console
  • Use integrated terminal for commands

Visual Studio Debugging

  • Full IntelliSense support
  • Advanced debugging tools
  • Performance profiling
  • Built-in test runner

Recommended Workflow

  1. Development: Use VS Code for cross-platform development
  2. C# Deep Work: Use Visual Studio for advanced C# features
  3. Testing: Run tests in both IDEs to ensure compatibility
  4. Deployment: Use AWS CLI and deployment scripts
  5. Monitoring: Use AWS Console and CloudWatch
