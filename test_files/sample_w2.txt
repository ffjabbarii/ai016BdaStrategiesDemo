How BDA Project works
Standard Output
Standard output is pre-defined extraction managed by Bedrock.

It can extract information from documents, images, videos, and audio. When creating a BDA Project, you can set your preference.

Custom Output
Custom output provides the flexibility to define extraction that are not offered from the standard output, it also allows you to transit information with a format that can merge into your data management system schema. To configure custom output, you need to add Blueprint(s) into a BDA Project as a custom output configuration.

Create Blueprint
If you do not find a corresponding blueprint from the existing blueprint list, you can create a blueprint. When you upload a file to create a new blueprint, the system will try to match from global categories. If the system cannot find a match, you can generate a blueprint in a minute

Add from Blueprint list
You can add sample or custom blueprints into a BDA Project. You can only extract files through blueprints of their respective modalities (For example, an image file can only be processed by an image blueprint).

Test & deploy
After creating, you can use the project in inference or test your project in console.

Manual modality routing (0) Info
Modality routing enables you to control how the uploaded files are processed by the project. By default, the system intelligently auto-routes the file to the most relevant modality. You can override default settings and specify how png, jpeg, mp4 or mov file type should be interpreted for data extraction. For example, a png file type can be treated as an document, or a mp4 file type can treated as an audio source.
